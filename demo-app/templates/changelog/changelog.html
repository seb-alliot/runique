Historique des corrections et améliorations apportées au framework et à cette application de démonstration.

v1.1.50 — 2026-03-20

- Fix — FileField
validation des restrictions

Fix : les fichiers invalides (mauvaise extension, taille dépassée, format image incorrect) sont maintenant supprimés du disque automatiquement si la validation échoue.
Fix : les soumissions sans fichier sélectionné (filename="") ne créent plus de fichier vide orphelin sur le disque — la contrainte required fonctionne correctement.
Fix : upload_to("chemin") applique maintenant le chemin exact. upload_to_env() utilise {MEDIA_ROOT}/{nom_du_champ}/. Le déplacement se fait dans finalize() uniquement si la validation passe.

- Feature — derive_form
option file()

Les modèles peuvent maintenant déclarer un champ fichier directement dans le DSL :
image: String [file(image, "media/uploads")]
derive_form génère automatiquement le FileField correspondant avec le bon type et le chemin d'upload. Types disponibles : image, document, any.

- Roadmap — refactorisation
système de champs en widget

L'option file() est une solution pragmatique. La direction long terme est une refactorisation du système de champs de formulaire en widget (à la Django) : chaque champ du modèle pourra déclarer son type de rendu indépendamment de son type SQL, sans cas particuliers dans le générateur.

v1.1.49 — 2026-03-19

- Bug —
is_valid()
-> bloquait tous les formulaires

Fix : suppression de set_expected_value() dans Forms::new()
Le CSRF est déjà validé en amont dans Prisme — la double validation était inutile et bloquante sur la validation des formulaires.
Ce qui invalidait les formulaires même quand les données étaient correctes.

- Fix —
makemigrations
-> ordre FK et updated_at

Les nouvelles tables sont triées topologiquement avant génération — les tables référencées par FK sont enregistrées en premier dans lib.rs.
La colonne updated_at génère ON UPDATE CURRENT_TIMESTAMP (MySQL) ou un trigger (PostgreSQL) selon DB_URL / DB_ENGINE.
Le diff détecte les colonnes qui ont gagné ou perdu DEFAULT CURRENT_TIMESTAMP et génère un ALTER automatiquement.
