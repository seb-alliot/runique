Validation des hôtes autorisés — protège contre les attaques Host Header. Supporte les wildcards, les ports explicites et les adresses IPv6.

main.rs — configuration via le builder
.middleware(|m| {
    m.with_allowed_hosts(|h| {
        h.enabled(true)
         .host("monsite.fr")
         .host("www.monsite.fr")
    })
})

// Wildcard sous-domaines
h.enabled(true).host(".monsite.fr")

// Port explicite
h.enabled(true).host("localhost:8080")

// IPv6
h.enabled(true).host("[::1]:8080")
Conditionnel selon l'environnement
m.with_allowed_hosts(|h| {
    h.enabled(!is_debug())  // désactivé en dev, actif en prod
     .host("monsite.fr")
     .host("www.monsite.fr")
})

// Sans appel à with_allowed_hosts → désactivé par défaut.
Fonctionnement
// Le middleware inspecte le header Host.
// Si l'hôte n'est pas dans la liste :
HTTP/1.1 400 Bad Request

// Wildcards supportés :
//   *.exemple.fr → api.exemple.fr ✓
//              → sous.api.exemple.fr ✗ (un seul niveau)

Extensions(0)
  → ErrorHandler(10)
  → Custom(20+)
  → Host(70)    // ici
  → routes
