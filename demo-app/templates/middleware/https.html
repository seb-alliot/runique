Redirection automatique HTTP → HTTPS et en-têtes de sécurité HSTS. Fonctionne via le header X-Forwarded-Proto — nécessite un reverse proxy.

.env — activation
# Activer la redirection HTTPS + HSTS
ENFORCE_HTTPS=true

# Redirige vers HTTPS si X-Forwarded-Proto: http
# Ajoute automatiquement le header HSTS.
En-têtes générés
// Redirection HTTP → HTTPS :
HTTP/1.1 301 Moved Permanently
Location: https://monsite.fr/...

// Header HSTS ajouté sur les réponses HTTPS :
Strict-Transport-Security:
  max-age=31536000; includeSubDomains
Prérequis — reverse proxy
// enforce_https lit X-Forwarded-Proto.
// Ce header doit être injecté par le proxy.

# nginx
proxy_set_header X-Forwarded-Proto $scheme;

# Caddy
# injecté automatiquement

// ⚠ Sans proxy de confiance, ce header
//   peut être forcé par un client.
//   Ne jamais l'activer en local pur.
Combinaison recommandée
// .env — activer HTTPS :
ENFORCE_HTTPS=true

// main.rs — combiner hosts + CSP strict :
use runique::app::builder::RuniqueAppBuilder as builder;
use runique::middleware::security::csp::SecurityPolicy;
use runique::prelude::RuniqueConfig;

builder::new(RuniqueConfig::from_env())
    .middleware(|m| m
        .with_allowed_hosts(|h| h
            .enabled(true)
            .host("monsite.fr"))
        .with_csp(|c| c
            .policy(SecurityPolicy::strict())))
    .routes(url::routes())
    .build().await?
    .run().await?
