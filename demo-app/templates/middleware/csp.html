Content Security Policy — contrôle quelles ressources le navigateur peut charger. Trois profils prédéfinis, directives personnalisables, support du nonce.

main.rs — profils prédéfinis
use runique::middleware::csp::SecurityPolicy;

RuniqueApp::new()
    .middleware(SecurityPolicy::default())
    .urls(routes())
    .run().await

// Profils disponibles :
//   SecurityPolicy::default()    — équilibré
//   SecurityPolicy::strict()     — aucune ressource externe
//   SecurityPolicy::permissive() — développement
Directives personnalisées
SecurityPolicy::default()
    .script_src(["'self'", "cdn.example.com"])
    .style_src(["'self'", "'unsafe-inline'"])
    .img_src(["'self'", "data:"])
    .connect_src(["'self'"])
Nonce — scripts inline sécurisés
{# template — injecter le nonce #}
<script nonce="{{ csp_nonce }}">
    console.log("script autorisé");
</script>

// Le nonce est régénéré à chaque requête.
// Il est automatiquement injecté dans le contexte Tera.
En-têtes générés
// Header envoyé au navigateur :
Content-Security-Policy:
  default-src 'self';
  script-src  'self' 'nonce-abc123';
  style-src   'self' 'unsafe-inline';
  img-src     'self' data:;

// Notes :
// — enforce_https active HSTS automatiquement
// — nécessite un reverse proxy de confiance
