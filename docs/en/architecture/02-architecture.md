# Architecture of a Runique Project

A Runique project is a standard Rust binary crate. `runique new` generates the base structure, which you evolve as needed.

## Project structure

```text
my-project/
тФЬтФАтФА src/
тФВ   тФЬтФАтФА entities/          # Model declarations
тФВ   тФВ   тФЬтФАтФА users.rs       # Uses Runique AST for the makemigrations CLI
тФВ   тФВ   тФФтФАтФА blog.rs        # CLI is not compatible with plain structs
тФВ   тФВ
тФВ   тФЬтФАтФА formulaire/        # Form declarations
тФВ   тФВ   тФЬтФАтФА inscription.rs # Using the form engine
тФВ   тФВ   тФФтФАтФА blog.rs        # or proc-attribute macros
тФВ   тФВ
тФВ   тФЬтФАтФА main.rs            # Entry point тАФ RuniqueApp builder
тФВ   тФЬтФАтФА admin.rs           # admin!{} declaration (if admin enabled, required for runique start)
тФВ   тФЬтФАтФА urls.rs            # urlpatterns! тАФ routing table
тФВ   тФЬтФАтФА views.rs           # Handlers (async functions)
тФВ   тФФтФАтФА forms.rs           # RuniqueForm structs (or forms/ folder)
тФВ
тФЬтФАтФА templates/             # Tera templates (.html)
тФЬтФАтФА static/                # Static files (CSS, JS, images)
тФВ   тФФтФАтФА media/             # Uploads (FileField)
тФВ                          # media/ can be placed in another folder
тФВ
тФЬтФАтФА migration/             # SeaORM migrations
тФВ   тФФтФАтФА src/
тФВ       тФФтФАтФА lib.rs
тФЬтФАтФА .env                   # Environment variables
тФФтФАтФА Cargo.toml
```

## Role of each file

**`main.rs`** тАФ Configures and starts the application via the builder:

```rust
#[macro_use]
extern crate runique;
use runique::prelude::*;

mod entities;
mod formulaire;
mod urls;
mod views;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let config = RuniqueConfig::from_env();
    let db = DatabaseConfig::from_env()?.build().connect().await?;

    password_init(PasswordConfig::auto());

    RuniqueApp::builder(config)
        .routes(urls::routes())
        .with_database(db)
        .statics()
        .build()
        .await?
        .run()
        .await?;

    Ok(())
}
```

**`urls.rs`** тАФ Declares routes via `urlpatterns!`:

```rust
use runique::prelude::*;
use crate::views;

pub fn routes() -> Router {
    urlpatterns! {
        "/" => view!{ views::index }, name = "index",
        "/register" => view!{ views::register }, name = "register",
    }
}
```

**`views.rs`** тАФ Request handlers:

```rust
pub async fn index(mut request: Request) -> AppResult<Response> {
    context_update!(request => { "title" => "Home" });
    request.render("index.html")
}
```

**`formulaire/`** тАФ Typed forms:

```rust
pub struct RegisterForm { pub form: Forms }

impl RuniqueForm for RegisterForm {
    fn register_fields(form: &mut Forms) {
        form.field(&TextField::text("username").label("Username").required());
        form.field(&TextField::email("email").label("Email").required());
    }
    impl_form_access!();
}
```

**`admin.rs`** тАФ Admin view declaration (files generated into `src/admins/`):

```rust
admin! {
    users: users::Model => RegisterForm {
        title: "Users",
        permissions: ["admin"],
    }
}
```

## Starting a new project

```bash
runique new my-project
cd my-project
runique start
```

`runique new` generates the minimal structure above. What to edit and what to leave alone:

| File / folder | Edit? | Role |
| --- | --- | --- |
| `src/main.rs` | Yes | Configures the builder and declares modules |
| `src/urls.rs` | Yes | Routing table |
| `src/views.rs` | Yes | Request handlers |
| `src/entities/` | Yes | Model declarations (Runique AST, compatible with `makemigrations`) |
| `src/formulaire/` | Yes | Forms and validation |
| `src/admin.rs` | Yes (if admin) | `admin!{}` declaration, required for `runique start` |
| `src/admins/` | **No** | Generated by the daemon тАФ do not edit manually |
| `templates/` | Yes | Tera templates |
| `static/` | Yes | CSS, JS, images |
| `migration/` | No (unless adding tables) | SeaORM migrations |
| `.env` | Yes | Environment variables |

> `runique start` watches `src/admin.rs` and regenerates `src/admins/` on every change. For apps without an admin view, `cargo run` is sufficient.

## Framework internal structure

```text
runique/src/
тФЬтФАтФА app/                    # App Builder, Templates & Intelligent Builder
тФВ   тФЬтФАтФА builder.rs          # RuniqueAppBuilder with slots
тФВ   тФЬтФАтФА error_build.rs      # Build errors
тФВ   тФЬтФАтФА templates.rs        # TemplateLoader (Tera)
тФВ   тФФтФАтФА staging/            # Staging structs
тФВ       тФЬтФАтФА core_staging.rs
тФВ       тФЬтФАтФА middleware_staging.rs
тФВ       тФФтФАтФА static_staging.rs
тФЬтФАтФА config/                 # Configuration & Settings
тФЬтФАтФА context/                # Request Context & Tera tools
тФВ   тФЬтФАтФА request.rs          # Request struct (extractor)
тФВ   тФФтФАтФА tera/               # Tera filters and functions
тФЬтФАтФА db/                     # ORM & Database
тФЬтФАтФА engine/                 # RuniqueEngine
тФЬтФАтФА errors/                 # Error handling
тФЬтФАтФА flash/                  # Flash messages
тФЬтФАтФА forms/                  # Form system
тФЬтФАтФА macros/                 # Utility macros
тФВ   тФЬтФАтФА context_macro/      # context!, context_update!
тФВ   тФЬтФАтФА flash_message/      # success!, error!, info!, warning!, flash_now!
тФВ   тФФтФАтФА router/             # urlpatterns!, view!, impl_objects!
тФЬтФАтФА middleware/             # Middleware (Security)
тФВ   тФФтФАтФА security/           # CSRF, CSP, Host, Cache, Error Handler
тФЬтФАтФА utils/                  # Utilities
тФЬтФАтФА lib.rs
тФФтФАтФА prelude.rs
```
